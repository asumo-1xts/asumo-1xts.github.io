# Pythonでツイート全削除2026夏

[<Badge type="tag" text="Python" />](../tags/python)

## はじめに

事情により自分の過去のツイートを根こそぎ削除する必要が出てきてしまい、外部サービス（「黒歴史クリーナー」的なやつ）を利用するのはセキュリティの面で気が引けたので自前の環境からX-APIを叩いて遂行しました。基本的にはインターネット上の情報を頼りにして何とかなったのですが、X-APIの仕様改変が激しすぎてリアルタイムに正しい記事が見当たらなかったので、ここに2026年6月時点での方法を記しておきます。

::: warning

X-APIは今年の2月からpay-per-use（従量課金制）になったらしく、これに伴って無料枠が消滅しました。すなわち利用に際して必ず料金が発生してしまいます。

ツイート削除の場合は1000件につき5ドル必要で、私は50ドル支払って約10000件のツイートを削除しました。私としてはギリッギリ許容範囲内でしたが、完全無料の方法をお探しの方は悪しからず。

:::

## 環境

- Debian 13 on WSL2
- Python 3.13.5

先んじて手元のPython環境で新規プロジェクト（今回は「post-deleter」と命名）を用意しておきましょう。私はuvを使っているので`uv init post-deleter`しました。

## アーカイブを取得

Xにて「設定とプライバシー」>「アカウント」>「データのアーカイブをダウンロード」と進んで、「アーカイブをリクエスト」します。

数日経つとZIPファイル（私の場合は数GBありました）が現れるので頑張ってダウンロードして、プロジェクト用ディレクトリ内で解凍します。フォルダ名はそのままだと長いので`post-archive`とでも改名しておきましょう。主に必要になるのは`data/tweets.js`です[^1][^2]。

## X Developer

### 開発者アカウントとアプリを作成

[X Developer](https://console.x.com/)にてXのアカウントと紐づいた開発者アカウントを取得します。続けて「新しいクライアントアプリケーションを作成」して、「Environment」は「Development」を選択します。

### 権限を設定

画面右上の「Settings」にて、各項目を以下のように設定します。

| 項目         | 値                                                                                                   |
| ------------ | ---------------------------------------------------------------------------------------------------- |
| アプリの権限 | 読み書きおよびダイレクトメッセージ                                                                   |
| アプリの種類 | ウェブアプリ、自動化アプリまたはボット                                                               |
| アプリ情報   | コールバックURL: `http://localhost:3000/` <br/> ウェブサイトURL: 適当（自分のXアカウントページなど） |

### APIキーを取得

今回はOAuth1.0が使えればよいので、

「OAuth 1.0 キー」以外は適当に無視してください。

このページを閉じると各キーの値は二度と確認できませんので（再生成はできます）、`.env`ファイルを作成して、それぞれコピペしておきます。ついでにアーカイブファイルのパスも加えました。

::: code-group

```ini [.env ~vscode-icons:file-type-dotenv~]
CONSUMER_KEY = ******
CONSUMER_SECRET = ******
ACCESS_TOKEN = ******
ACCESS_TOKEN_SECRET = ******

archive_file_path = post-archive/data/tweets.js
```

:::

## Pythonスクリプト

お手元のPython環境にて、ここまでに登場したファイルやフォルダを以下のように配置してください。

```
tree
```

続けて`main.py`と同じ階層に`myfunc.py`を新規作成して、それぞれ以下の通りにします。

::: code-group
<<< @/snippets/2026/18-main.py{python} [main.py]
<<< @/snippets/2026/18-myfunc.py{python} [myfunc.py]
:::

あとは`main.py`を実行するだけです。私の試した限りでは、ツイートの削除は一回のリクエストにつき最大50件までしか処理されず、また一度リクエストを行ったら15分ほどクールタイムが必要です。これらの条件を盛り込んでかなり長丁場のスクリプトになっているので、実行したあとはしばらく放置しておくのが良いです。

それから、このスクリプトでは特定の年のツイートを検索・抽出して削除するようにしており、少し手を加えれば詳細な期間設定が可能です。きちんと調べていませんが、X-APIを使いこなせば「リプライや画像ツイートは削除しない」みたいな条件設定もできるかもしれません。
