## はじめに

### 動機

JIS配列のREALFORCEのスライダーをCherry MX軸に総交換して、Windows側の設定でUS配列キーボードとして認識させて

- US配列
- ただしEnterキーはISOタイプ
- 変換・無変換キー（実際にはLANG1・LANG2キー）有り

として2年ほど愛用してきたのですが、これの分割バージョンが欲しくなりました。

## ファームウェア

幸いにも[冒頭](#教材)で紹介した教材では分割キーボードがお手本になっていたので、QMK周りであまり躓かずに済みました。

ささやかな自分なりのカスタマイズとして、`keymap.c`にて以下の機能を追加しました。

- 現在のレイヤーを2つのLEDの点灯状況でインジケート
- Mキーをを押しながら電源を入れるとMIDIコントローラ化する

::: code-group

```c [keymap.c ~vscode-icons:file-type-c~]
...

// 電源投入時に走る関数
void keyboard_post_init_user(void) {
    setPinOutput(LED_LAYER_1);
    setPinOutput(LED_LAYER_2);
    flash_led(100);
    matrix_scan();
    // 8行2列のMキーが押されているかを確認
    if (matrix_get_row(8) & (1 << 2)) {
        set_single_default_layer(2);    // レイヤー2をデフォルトに設定
    }
}

// デフォルトレイヤーが変わったときに呼ばれる関数
layer_state_t default_layer_state_set_user(layer_state_t state) {
    if (get_highest_layer(state) == 1) {
        writePinHigh(LED_LAYER_1);
        writePinLow(LED_LAYER_2);
    } else if (get_highest_layer(state) == 2) {
        writePinLow(LED_LAYER_1);
        writePinHigh(LED_LAYER_2);
    } else {
        writePinLow(LED_LAYER_1);
        writePinLow(LED_LAYER_2);
    }
    return state;
}

...
```

## PCB基板

キースイッチのフットプリントは有志の方々がGitHubに公開してくれているものを使うと良いのですが、如何せん（自作キーボードのコミュニティが広大なおかげで）選択肢が複数あって迷ってしまうかもしれません。私が試した限りでは、「PCBエディタのグリッド間隔を1/8 u (= 19.05/8 mm)にしたとき確実にぴったり並んでくれる」という点において[koktoh](https://github.com/koktoh 'koktoh')氏の公開しているライブラリが最も良かったです。

[BrownSugar Keyboard KiCad Library](@:https://github.com/koktoh/BrownSugar_KBD_KiCad_Library)

Cherry MXシリーズしか収録されていないのでKailh Chocシリーズ等を使う場合には他のライブラリを探す必要がありますが、それでも一旦はこのライブラリのフットプリントで並べておいて後から置換すると確実でした。

## 筐体

### 設計

当初は基板プレート3枚による単純なサンドイッチ方式を考えていたのですが、周囲にスペーサーを積層して隙間を無くしたいなと思い直し、どうせならプレートと周囲のスペーサーを一体化して3Dプリントするのが良いのでは？というところに落ち着きました。

私は3DCAD全般があまり好きでないので、KiCadで各階層のパーツを作ってSTEPファイルに書き出してから、それらのアセンブルだけFreeCADで行いました。A2plusというワークベンチを利用して面と穴の拘束条件を2つほど与えれば、所望のアセンブリが完成します。

### 発注

各アセンブリは最終的に以下のようになりました。

| アセンブリ                 | 役割 / 備考                                                                                                                                                 | 厚み [mm] |
| -------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------- | --------- |
| 表面プレート + スペーサー① | キースイッチをはめ込み、PCB基板までの隙間を埋める。<br/>無いことも多いと思われる                                                                            | 1.2 + 1.0 |
| （PCB基板）                | 最低限これさえあれば良い                                                                                                                                    | 1.6       |
| スペーサー② + 裏面プレート | 裏面プレートまでの隙間を埋める。<br/>Arduino基板やTRS端子が収まるよう厚めにとっている。裏面プレートはあまり厚くしすぎるとフィラメント代が露骨に嵩むので注意 | 6.3 + 2.1 |

これらの固定方法について、今回はM2ねじでセルフタップすることにしました。表側のアセンブリには直径2.0 mmの通し穴を、裏側のアセンブリには直径1.6 mmの下穴を開けています。
